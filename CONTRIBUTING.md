# Contributing

Thanks for your interest in improving This project.

## How to contribute
- Open an issue to describe the bug or idea.
- Keep each pull request focused on one change.
- Use clear commit messages and small, reviewable updates.
- Add or update tests when behavior changes.
- Update documentation when commands, setup, or behavior changes.

## Local setup
1. Fork and clone the repository.
2. Run `npm install`.
3. Add a local `.env` file for required keys.
4. Run `npm start` and test your change.

## Code style
- Prefer simple names and short functions.
- Keep comments direct and easy to understand.
- Avoid unnecessary abstractions in this learning-focused codebase.

## Pull request checklist
- [ ] Change works locally
- [ ] Documentation is updated
- [ ] No sensitive information is committed
